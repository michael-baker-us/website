Problems I solve

Engineering experiments

The lab.

Most of my track record lives behind a company firewall — the CI platform ~200 repos run on, the migrations, the internal tools. You can't star any of it on GitHub. The lab is the opposite: small public builds slightly beyond my reach, each chosen for what it teaches me. Shared early, judged on learning — not polish.

quiver

shipped

A Git-friendly API client and collection runner — a Postman replacement you can code-review. Plain YAML collections, a CLI that drops into CI, OpenAPI import, k6 export, and a local web UI.

teaches: CLI design · HTTP internals · OpenAPI · CI integrations · monorepo architecture

git-branch-cleaner

shipped

A CLI (gbc) that audits a repo's local branches — merged, gone, stale, abandoned — and cleans them up safely. Read-only by default, archives every branch as a recovery tag before deleting, and never touches protected or remote branches.

teaches: CLI design · git plumbing · safe-by-default UX · TypeScript packaging · testing

ci-pipeline-visualizer

shipped

Turns GitHub Actions and GitLab CI YAML into one interactive dependency graph. Both dialects reduce to a single common model, then render as a self-contained HTML page — job dependencies, stages, artifact flow, and (with run data) timings, the critical path, and bottlenecks.

teaches: parser design · graph layout · static analysis · self-contained rendering · TypeScript

git-galaxy

shipped

Turns any git repo's history into an animated 3D galaxy — commits become stars, folders become planets, files become satellites — explorable in the browser, with a flyable spaceship for good measure.

teaches: Three.js · procedural layout · git plumbing · shader uniforms · monorepo architecture

OutRun Racer

building

A pseudo-3D arcade racer in the style of 1986. Because nothing teaches performance like a render loop.

teaches: rendering · game loops · camera systems · optimization

on the bench: deployment planner · config diff viewer · release notes generator · log explorer · rest mock server · docker image explorer · env var manager

Working with me

Operating principles.

  1. /01

    Systems before symptoms

    The third time the same thing breaks, the problem isn't the thing — it's the system around it. I fix the system.

  2. /02

    AI is leverage, not a shortcut

    I use AI to move faster on work I understand deeply — never to ship things I couldn't defend in review.

  3. /03

    Ship the boring fix

    The clever solution impresses; the boring one survives. I optimize for the engineer who inherits my work.

  4. /04

    Leave a paper trail

    Every fix comes with the write-up: what broke, why, and how to see it coming next time. Docs are part of done.
